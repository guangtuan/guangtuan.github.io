android 动态加载 so 库学习笔记

动态加载 so 库

关于动态加载 so 库，这篇文章 有很详细的讲解，这里只贴出部分学习笔记。

System 类有两个加载 so 库的方法

  • load(String pathName) 从 so 库的路径加载 so 库
  • loadLibrary(String libName) 根据 so 库的名字加载 so 库（系统在安装应用的时候，扫描包内是否存在 so 库，并且复制到对应的标准的内部存储路径中，所以可以根据 so 库的名字加载）

所以一个可行的动态加载 so 库的思路是，从网上下载需要的 so 库，复制到内部存储（外部存储中的文件没有可执行的权限），再使用 loadLibrary(String libName) 方法加载 so 库。

为使用 Realm 的应用瘦身

Realm 是一个工作在 navtive 层的帮助开发者对数据进行本地存储的框架。

虽然工作在 native 层给程序运行的效率带来了很大的提高，但因为需要支持不同的 CPU 架构，所以引用了 Realm 的应用的包体积，也不可避免的增大了许多。

我们可以看到，解压安装包以后，lib 目录下存在六份 so 文件，但是

Android 包管理器安装 APK 时，会自动选择 APK 包中为对应系统 ABI 预编译好的 .so 文件，如果在对应的 lib/ABI 目录中存在 .so 文件的话。

所以，实际上安装包上的六个 so 库，对于用户的手机来说，只有一个是会被安装，并且起作用的。

那么，其他的五个 so 库实际上是多余的，如果能够从安装包中去掉多余的五个 so 库，就可以达到为 APK 瘦身的目的。

我们如何得知系统支持何种 ABI 架构？使用 Android 自带的 API 即可。

我们再来看 Realm 是如何加载 so 库的。

查看 Realm 的源码，io.realm.internal.Realm$loadLibrary

我们可以在自定义 Application 中先于 Realm 框架

  • 获取系统支持的 ABI 架构
  • 下载我们需要的 so 库，复制到内部存储，使用动态加载的办法加载 so 库中
  • 使用反射修改 libraryIsLoaded 变量的值为 true，避免二次加载
  • 手动调用 Version.coreLibVersionCompatible(true); 方法

这样子，在编译过后的 APK 中，我们可以将对应的 so 库都删除掉，在用户首次使用应用的时候，使用动态加载的方法加载 so 库，从而达到为应用瘦身的目的。
