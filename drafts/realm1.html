BaseModel 的存在

实际使用中，使用 BaseModel 进行多重继承在 Realm ９.0 或者更早版本中行不通，这里删去。但是思想保留，万一哪天支持了呢。

Realm 中要求所有的 Model 继承于 RealmObject, 但是我们不能避免以后会使用别的替代方案，所以一个 BaseModel 的存在是很有必要的，自身工程里的 Model 全都继承自 BaseModel, 只需要让 BaseModel 继承自 RealmObject 即可。

通用的 Dao

Dao, Data Access Object, 在 Java 的 web 开发中很常见，但是在 Android 开发上，对于每一个 Model 都写一个 Dao 似乎有点繁琐。

如果在 Model 中封装一些常用的增删改查操作的话，就会对 Realm 产生依赖。但是考虑到解耦的原则，这样做并不好。

因此，通用的 Dao 是有必要存在的，如何体现 Dao 所操作的 Model, 这里我的思路是使用泛型。
