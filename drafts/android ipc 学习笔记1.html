AIDL 概览

AIDL 的全称是 Android Interface Definition Language Android 接口定义语言。

在编译的过程中，开发工具比如 Android Studio 会根据 AIDL 文件的声明来自动生成对应的接口，使用 Android Studio 建立一个 AIDL 文件。

DemoInterface.aidl

AIDL 的语法细则 支持的数据类型

从 basicTypes 方法的注释里可以明白，AIDL 声明的接口，允许将 int long boolean float double String 这六种类型的变量作为参数以及返回值，实际上还一些没有提到的数据类型

  • 所有基本数据类型，以及 CharSequence
  • 任何实现了 Parcelable 接口的类
  • ArrayList< T supported by AIDL >
  • HashMap< T supported by AIDL, T supported by AIDL >
  • AIDL, AIDL 接口本身也在 AIDL 文件中可以被使用

AIDL 的语法细则 导包

如果在 AIDL 中使用了某个实现 Parcelable 接口的 Model，我们假设 Model 的定义如下：

生成的代码

在 moduleName/build/generated/source/aidl/debug/com/packageName/ 中我们可以看到 Android Studio 为我们生成的接口代码，结构如下：

  • DemoInterface
    • 接口方法 basicTypes()
    • 抽象类 Stub 继承自 Binder，实现了 DemoInterface 接口
      • 内部静态类 Proxy 实现了 DemoInterface 接口

Stub 的结构如下

  • Stub extends Binder
    • 静态常量 DESCRIPTOR 一般为类名
    • asInterface() 将服务端的 Binder 对象转换为客户端需要的 AIDL 接口的实例，如果客户端和服务端在同一个进程，返回服务端的 Stub 对象本身，否则返回 Proxy 对象
    • asBinder() 返回 Binder 对象本身
    • onTransact() 方法，这个方法的参数如下，看到这个方法，终于有点 server and client 的味道了
      • int code 表示需要调用哪个方法，IDE 会为每一个接口方法都生成一个整形的标识符
      • Parcel data 方法的参数从 data 中读出
      • Parcel reply 方法的返回值写入 reply 中
      • int flags 没发现是做啥的
      • 返回值：返回 true 客户端请求成功，否则返回 false，从安全的角度来讲，可以用来做权限验证，对于未知来源的请求，返回 false 即可拦截。
