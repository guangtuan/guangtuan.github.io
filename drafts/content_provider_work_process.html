这篇文章为读书笔记，主要内容是 ContentProvider 的启动过程。

方法入口

进程启动

AMS 启动一个进程的时候调用了 startProcessLocked() 方法，在这个方法内有一段代码。

这段代码中调用了 Process#start() 方法，加载了类 ActivityThread，我们知道 java 程序的入口是静态的 main 方法，这在 Android 中也一样。

ActivityThread#main()

ActivityThread#attach()

在 ActivityThread 的 attach 方法中，调用了 IActivityManager#attachApplication() 将 AMS 和 Application 绑定起来.

  • AMS 的 attachApplication() 方法实际调用了 attachApplicationLocked() 方法

  • 继而调用了 IApplicationThread#bindApplicaion()，

  • 接着通过发送一个 Message 对象到内部的 Handler 的子类 H，

  • 调用了 ActivityThread#handleBindApplication()，在这个方法里完成了 Application 和 ContentProvider 的创建。

installProvider() 方法的详情，通过类加载器加载了相应的 ContentProvider

ContentProvider#attachInfo() 方法的详情，在这里设置了 ContentProvider 的编写者在 manifest 文件中设置的一些属性，并且调用了 onCreate() 方法。

另外，在 installProvider() 方法中，用到的 provider 对象是 IContentProvider 类型，和 ContentProvider 对应，与 BroadcastReceiver Service 等相类似。因此 ContentProvider 的 crud 方法都是跨进程地工作。
