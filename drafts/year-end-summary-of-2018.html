年终总结

跳槽

年初从 vanchu 离职，朋友内推到 yiwill，安卓岗位入职，不过做的多是前端的工作，渐渐开始思考自己想要做的是什么样的工作，作为一个前端，感觉写界面是无尽的折磨，更偏爱逻辑的处理。

再次跳槽

6 月份，根据自己平时折腾 node 和 vps 的经验再次跳槽，找到了一个初创公司 node 后端的职位，这是一次赌博，不过目前看来我赌到了我想要的，甚至比我想要的更多。

混乱是阶梯

公司初创，后来因为 node 招人困难，开始全面使用 java，但是开发普遍年轻，项目一片混乱。

在 app 项目 2.0 启动的时候，领导在矮个里挑高个，让我和另一个同事做了技术主管。

当时的想法是，如果没有足够的决心离开，就应该尽自己的努力，将团队的运转带上正轨，所以我做了一些改变，把三个月的工作成果 – 用 node 写的通道项目果断舍弃了，交给另一个主管用 java 重新开发，并且全面投入 Spring boot 的怀抱当中，加入了 app 的项目组。

混乱是阶梯，主管这个职位虽然很虚，而且让人承担了更多的责任，但带来的好处是，一些决策自己可以起到作用。

战术的力量

团队是需要战术的，规范化的开发流程是项目稳定必不可少的条件之一，几个月来我都在和项目经理一起尽力推动这个事情，目前大家已经能做到 需求评审 -> 接口协议 -> 前后端并行开发 -> 测试 -> 上线 这样一套流程，不说百分之百，七八成的执行力还是有的，在团队人员综合能力普遍较低的情况下，能够做到这样已经很不错了。

较之于早期，这规避了很多出错的情况，节省了很多时间，也少了一些无意义的加班。

个人成长

Coding 技能上有很大的增长，学到了很多东西。

  • 前期兼职运维管理服务器，虽然 awk sed grep 还没有到信手拈来的程度，但查一查都能用。

  • 一直在学习 docker 的使用，因为项目的体量不够，还没有上 k8s，但是已经在稳定的运转 jenkins 构建 -> 上传镜像私服 -> 生产机拉取镜像运行 这一套流程。

  • 已经会熟练的使用 Spring Boot，算是一名合格的 CURD boy 了。

  • 前端方面尝试了一下 react js，相对于 vue，它给我的感觉是全新的，我觉得前端就应该这么写，稍微写了一个模块，尝试了很多函数式编程的做法，很满足。
